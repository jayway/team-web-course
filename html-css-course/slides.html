Headlines

  • Each document should have a logical heading structure that starts at level 1 and doesn’t skip any levels.
  • The title of the document’s main content should be a level 1 heading.
  • There should be only one level 1 heading in each document section (HTML5).

Divs

Divs are used to group content for layouting and positioning with CSS. 

<div>
    <h3>This is a heading</h3>
    <p>This is a paragraph.</p>
</div>

Divs could be nested. This is useful if you are using divs to layout a web page.

<div class="container">
    <div class="content-primary">Main content area</div>
    <div class="content-secondary">Secondary content area</div>
</div>

Spans

SPAN carries no structural meaning itself, but it can be used to provide extra structure through its LANG, DIR, CLASS, and ID attributes.

<p>A text which is unstyled <span class="stand-out">except from this part</span></p>

Images

Used to display an image on a web page.

<img src="../image.png" alt="The Jayway logotype">
  • The src (source) attribute points out the URL to the image to display.
  • The required alt (alternative) attribute specifies an alternate text for an image, if the image cannot be displayed.

Links

A link, when clicked (or tapped), takes you to another document or to a section within the current document.

<a href="http://www.jayway.com" rel="start">This is a link</a>

Links

The id attribute can be used to create a bookmark inside the HTML document.

<a id="detailed-info">This is a document internal anchor</a>

And here are two ways of linking to the anchor.

<a href="#detailed-info">Click to read more</a>
<a href="http://www.jayway.com/page-with-anchors.html#detailed-info">Click to read more on jayway.com</a>

CSS

Cascading Style Sheet

Where is it defined?

You can have your CSS:

  • inline in your tags (not good practice)
    • <p style="color: blue">Inline example</p>
  • within a style element in your HTML
    • <style type="text/css">
  .banner { border: 1px solid #aaa; }
  .header { padding: 1em; }
</style>
  • or in a separate CSS file (best practice)
    • <link rel="stylesheet" href="style.css">

Styling

Selectors define the rule for what element(s) in your HTML document you want to target for styling.

There are different kinds of selectors. You can select by:

type, id, class, universal, descendant, child, adjecent sibling, pseudo elements and pseudo classes.

You can also group your selectors.

p span.highlight { font-weight: bold; color: red; }
h1, a:hover { text-decoration: underline; }
li a { text-decoration: none; }
body > p { margin: 0 10px; }
h1 + h2 { margin-top: -5mm }

The Type Selector

A type selector matches every instance of an element type in the document tree.

h1 { font-family: sans-serif; } 
ul { list-style-type: none; }

The ID Selector

No two id attributes can have the same value, they are unique within the HTML document.

<h1 id="chapter1">Having a Good Time!</h1>

Styling the element by id.

#chapter1 { font-weight: bold; }

The Class Selector

Classes are not unique and thus permits you to style multiple elements with the same properties.

<li class="menu-item">Book Index</li>
<li class="menu-item">Chapter 1</li>
<li class="menu-item">Chapter 2</li>
<li class="menu-item">Chapter 3</li>
<li class="menu-item">Chapter 4</li>

Styling the element by class.

.menu-item { padding: 5px; }

Pseudo Classes

To style elements based on user interaction or information that lies outside the document tree, pseudo classes can be used.

For example if a link has been visited or to style the first paragraph child of a div tag.

a:visited { color: blue }
div > p:first-child { text-indent: 0 }

Chained Selectors

#header .menu-item { text-transform: uppercase; }

This selector targets all menu items in the header list below but not those in the footer list.

<div id="header">
    <ul>
        <li class="menu-item">Item 1</li>
        <li class="menu-item">Item 2</li>
        <li class="menu-item">Item 3</li>
    </ul>
</div>
<div id="footer">
    <ul>
        <li class="menu-item">Item 1</li>
        <li class="menu-item">Item 2</li>
        <li class="menu-item">Item 3</li>
    </ul>
</div>

Stacked Selectors

p.highlight { background-color: #eaeaae; }

This selector targets the <p> element with a class attribute value set to 'highlight'.

<p class="highlight">This paragraph is highlighted!</p>
<p>This <span class="highlight">This</span> remains unstyled.</p>

Grouping Selectors

To reuse and have the same styling declaration for a group of selectors you simply separate these selectors with a comma ','.

h1#title, a:hover { text-decoration: underline; }

The Universal Selector

The universal selector, written as an asterisk (*), matches the name of any element type.

.warning * { 
  border: 1px solid yellow; 
  background-color: #eecc00; 
}

The above example matches any element inside an element with the class “warning”.

<div class="warning">
  <p>Hey, watch out!</p>
  <div>Outside paragraph.<p>Another story...</p></div>
</div>

The Box Model

Every element in the web is a rectangular box.

This is how the size of the box and the space around it is calculated.

the_box {
  width: 300px; 
  padding: 20px; 
  border: 20px solid green; 
  margin: 20px 30px 30px 20px; 
}

Layouting

Layouting in CSS is about distributing sections of content in different ways on the page.

This is usually done by using <div>:s. You can achieve a lot by setting width, margins, floating, using different positionings and display styles.

Inline vs block display

For so called phrase content (what makes up a paragraph) the default display type is 'inline'. For flow content it is 'block'. The difference is that block displayal (by default) has a width of 100%, that is, it automatically stretches horizontally within its' container.

Positioning

The position property enables you to place an element exactly where you want it on a webpage. There are four valid values:

  • Static (defalut value)
  • Fixed
  • Relative
  • Absolute

Static Positioning

Using the 'static' value (which is default) the element is positioned exactly where it would be as without styling.

#div-1 {
   position:static;
}

Fixed Positioning

Used to position an element on a fixed position relative to the browser window, even when scrolling. In this case the #div-1 block will be stuck at the bottom of the browser viewport.

#div-1 { 
    position: fixed; 
    left: 0px; 
    bottom: 0px; 
}

Relative Positioning

If you specify position:relative, then you can use top or bottom and left or right to move the element relative to where it would normally occur in the document.

#div-1 {
    position:relative;
    top:20px;
    left:-40px;
}

Absolute Positioning

When you specify position:absolute, the element is removed from the document and placed exactly where you tell it to go.

#div-1a {
    position:absolute;
    top:0;
    right:0;
    width:200px;
}

Combined Positioning

If we set relative positioning on div-1, any elements within div-1 will be positioned relative to div-1. Then if we set absolute positioning on div-1a, we can move it to the top right of div-1.

#div-1 {
    position:relative;
}
#div-1a {
    position:absolute;
    top:0;
    right:0;
    width:200px;
}

Floating Columns

If we float one column to the left, then also float the second column to the left, they will push up against each other.

#div-1a {
    float:left;
    width:150px;
}
#div-1b {
    float:left;
    width:150px;
}

Clearing Floats

Then after the floating elements we can "clear" the floats to push down the rest of the content.

#div-1a {
    float:left;
    width:190px;
}
#div-1b {
    float:left;
    width:190px;
}
#div-1c {
    clear:both;
}

Centering Content

Now, if we want to center div-1a and div-1c horizontally this can be done by setting its right and left margin widths to "auto". 

#div-1a, #div-1c { 
    margin: 0px auto; 
    width: 200px; 
}

The Cascade - Inheritance

Inheritance is the process by which properties are passed from parent to child elements even though those properties have not been explicitly defined by other means.

The Cascade - Inheritance

Why is this a good thing? Consider the following code:

div {
  font-size: 20px;
}
<div>
  <p>
    This <em>sentence</em> will have a 20px
    <a href="#">font-size</a>.
  </p>
</div>

If inheritance wasn’t at work in the above code, we’d have to specify a font-size declaration for each element in turn, to make sure that all the content in the sentence was rendered at 20px.

The Cascade - Inheritance

Some properties are automatically inherited to an element from its' parent. Example of these are:

font-family, font-size, color, background-color.

Though, if you want to manually inherit, for example, a parent elements' border, this is how to do it:

.parent { border: 5px solid #666; padding: 10px; }
.child { bor    der: inherit; }

The Cascade - Specificity

Specificity is a mechanism within the CSS cascade that aids conflict resolution. In order of importance:

  1. Existance of style attribute
  2. Counting id selectors
  3. Counting class, attribute and pseudo selectors
  4. Counting element type selectors and pseudo-element

Specificity Example

<p class="message" style="color:red;">Red Text</p>
In order of importance, from left to right:
Selector Inline Style IDs Classes, Attributes and Pseudo-classes Element Types and Pseudo-elements
Inline Style 1 0 0 0
body#home div#warning p.message { color: blue; } 0 2 1 3
* body#home>div#warning .message { color: red; } 0 2 1 2
#home #warning p.message { color: yellow; } 0 2 1 1
#warning p.message { color: white; } 0 1 1 1
p.message { color: green; } 0 0 1 1

Avoid Inline Styling

Inline style rules always have the highest specificity, the only way to overwrite them within the CSS cascade is to use the !important statement on the relevant declarations—an approach that creates a maintenance nightmare.

<p style="color:red;">Red Text</p>
p {
  color: blue !important;
}

Media Queries

You can apply different styles depending on a number of so called media queries.

/* Type */
@media screen, projection { ... }

/* Dimension */
@media (min-width: 780px) { ... }
@media (min-device-width: 640px) { ... }

/* Orientation */ 
@media (orientation: portrait) { ... }

/* Color */ 
@media (color) { /* Screen is in color */ }
@media (min-color-index: 256) { /* At least 256 colors */ }
@media (monochrome) { /* Screen is monochrome */ }
